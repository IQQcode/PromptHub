## 二、PromptGo 核心功能清单

### 2.1 Prompt 展示模块

#### 2.1.1 卡片式展示

**卡片字段定义**：
| 字段 | 类型 | 必须 | 说明 |
|-----|------|-----|------|
| id | string | ✅ | 唯一标识符 |
| title | string | ✅ | 标题（限 50 字符） |
| description | string | ✅ | 描述摘要（限 100 字符） |
| content | string | ✅ | 完整 Prompt 内容 |
| category | string | ✅ | 一级分类 |
| tags | string[] | ✅ | 标签列表（最多 5 个） |
| models | string[] | ❌ | 适配模型（默认"通用"） |
| language | enum | ✅ | zh / en / both |
| variables | object[] | ❌ | 解析出的变量列表 |
| useCount | number | ❌ | 使用次数（复制次数） |
| favoriteCount | number | ❌ | 收藏次数 |
| authorId | string | ✅ | 作者 ID |
| createdAt | datetime | ✅ | 创建时间 |
| updatedAt | datetime | ❌ | 更新时间 |

**展示布局**：
- PC（≥1200px）：4 列网格
- 平板（768px-1199px）：2 列网格
- 手机（<768px）：1 列网格
- 卡片间距：16px
- 卡片圆角：8px

**排序方式**：
1. 最新发布（默认）
2. 最多使用
3. 最多收藏

#### 2.1.2 详情页

**页面元素**：
- 标题（大号字体）
- 完整 Prompt 内容（代码块样式，支持语法高亮）
- 变量填充表单（如有变量）
- 一键复制按钮（复制填充后的完整内容）
- 收藏按钮（显示收藏数）
- 作者信息（头像、昵称）
- 发布时间 / 更新时间
- 标签列表（可点击跳转筛选）
- 适配模型标签
- 返回列表按钮

### 2.3 用户上传模块

#### 2.3.1 上传表单

**表单字段**：
| 字段 | 类型 | 必填 | 验证规则 |
|-----|------|-----|---------|
| title | text | ✅ | 1-50 字符 |
| description | textarea | ✅ | 1-200 字符 |
| content | textarea | ✅ | 1-5000 字符 |
| category | select | ✅ | 从分类列表选择 |
| tags | multi-select | ✅ | 1-5 个标签 |
| models | multi-select | ❌ | 可多选，默认"通用" |
| language | radio | ✅ | zh / en / both |

**表单交互**：
- 实时字数统计
- 字段验证提示（输入时校验）
- 标签输入支持自动补全
- 分类切换时推荐相关标签

#### 2.3.2 变量模板功能

**语法规范**：
```
{{变量名}}
{{变量名:默认值}}
{{变量名:默认值:描述}}
```

**示例**：
```
请帮我写一篇关于{{主题}}的文章，字数约{{字数:500}}字，风格为{{风格:专业:可选：专业/轻松/幽默}}。
```

**解析规则**：
1. 正则匹配：`/\{\{([^}]+)\}\}/g`
2. 解析结果：`{ name: string, default?: string, description?: string }`
3. 自动去重

**填充交互**（详情页）：
1. 检测到变量时，在 Prompt 内容下方显示填充表单
2. 每个变量对应一个输入框，显示变量名和描述
3. 有默认值的变量预填充默认值
4. 实时预览替换后的 Prompt
5. 点击"复制"时复制替换后的完整内容

#### 2.3.3 发布流程

```
┌─────────────┐     ┌─────────────┐     ┌─────────────┐     ┌─────────────┐
│  填写表单    │ ──▶ │  预览效果    │ ──▶ │  直接发布    │ ──▶ │  进入列表    │
└─────────────┘     └─────────────┘     └─────────────┘     └─────────────┘
                           │
                           ▼
                    ┌─────────────┐
                    │  修改调整    │
                    └─────────────┘
```

**预览功能**：
- 模拟卡片样式展示
- 显示解析后的变量列表
- 可返回修改

### 2.4 用户账户系统

#### 2.4.1 登录/注册

**注册方式**：
1. 邮箱注册（邮箱 + 密码 + 验证码）
2. 第三方登录：
   - GitHub
   - Google
   - 微信（可选）

**登录方式**：
1. 邮箱 + 密码
2. 第三方授权
3. 记住登录状态（7天）

**用户信息**：
| 字段 | 说明 |
|-----|------|
| id | 用户唯一 ID |
| email | 邮箱（唯一） |
| nickname | 昵称（可修改） |
| avatar | 头像 URL |
| createdAt | 注册时间 |

#### 2.4.2 个人中心

**我的收藏**：
- 收藏的 Prompt 列表
- 支持取消收藏
- 按收藏时间排序

**我上传的**：
- 个人上传的 Prompt 列表
- 支持编辑 / 删除
- 显示每个 Prompt 的统计数据

**使用统计**：
- 总上传数量
- 总被复制次数
- 总被收藏次数
- 近 7 天趋势图（可选）

**账户设置**：
- 修改昵称
- 修改头像
- 修改密码
- 绑定/解绑第三方账号

### 2.5 API 开放能力

#### 2.5.1 接口概览

**公开接口（无需认证）**：
| 接口 | 方法 | 说明 |
|-----|------|------|
| `/api/prompts` | GET | 获取 Prompt 列表 |
| `/api/prompts/{id}` | GET | 获取单个 Prompt |
| `/api/categories` | GET | 获取分类列表 |
| `/api/tags` | GET | 获取标签列表 |
| `/api/search` | GET | 搜索 Prompt |

**授权接口（需 API Key）**：
| 接口 | 方法 | 说明 |
|-----|------|------|
| `/api/prompts` | POST | 创建 Prompt |
| `/api/prompts/{id}` | PUT | 更新 Prompt |
| `/api/prompts/{id}` | DELETE | 删除 Prompt |
| `/api/user/favorites` | GET | 获取收藏列表 |
| `/api/user/favorites/{id}` | POST | 添加收藏 |
| `/api/user/favorites/{id}` | DELETE | 取消收藏 |

#### 2.5.2 API Key 机制

**申请流程**：
1. 用户登录后进入 API 文档页
2. 点击"申请 API Key"
3. 系统生成唯一 Key
4. 用户可查看/重置 Key

**使用方式**：
```
Authorization: Bearer {api_key}
```

**限流策略**：
- 公开接口：100 次/分钟/IP
- 授权接口：1000 次/分钟/Key

### 2.6 多语言支持

**界面语言**：
- 中文（zh-CN）
- 英文（en-US）
- 切换方式：顶部导航栏语言开关
- 默认：跟随浏览器 `navigator.language`

**Prompt 内容语言**：
- 上传时标注语言（中文/英文/双语）
- 筛选时可按语言过滤

**国际化实现**：
- 前端使用 i18n 方案
- 文案存储：`locales/zh-CN.json`、`locales/en-US.json`

## 五、总结

PromptGo 在参考 aishort.top 的基础上，增加了以下核心差异化功能：

1. **变量模板功能**：支持 `{{变量}}` 语法，提供填充表单，提升复用效率
2. **完善的用户系统**：收藏、上传管理、使用统计
3. **API 开放能力**：RESTful 接口，支持第三方集成
4. **模型适配标注**：明确标注 Prompt 适用的 AI 模型
5. **站内直接投稿**：无需跳转 GitHub，降低用户门槛

这些功能将使 PromptGo 成为一个更专业、更易用的 Prompt 管理平台。
